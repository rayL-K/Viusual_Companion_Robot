# VeyraLux 品牌门户创意与内容方向

> 目标：把 `veyralux.org` 建成高级、克制、具有空间感和生命感的品牌介绍站。它负责解释 VeyraLux 的方向、代表作品和工程原则，不承载摄像头、麦克风或实时陪伴主链路，也不把 V2 的开发目标包装成已经验收的产品能力。

## 1. 站点角色与域名边界

三个域名必须有稳定、互不混淆的角色：

| 域名 | 角色 | 页面必须使用的状态表述 | 禁止表述 |
| --- | --- | --- | --- |
| `veyralux.org` | VeyraLux 品牌门户、作品介绍、团队与开源入口 | “品牌门户”“代表作品”“研发方向” | “实时控制台”“当前机器人服务” |
| `robot.veyralux.org` | V1 评审/兼容演示入口 | “V1 演示”“比赛评审版本”“历史兼容入口” | “V2”“最新生产架构” |
| `anima.veyralux.org` | V2 开发入口 | “V2 开发中”“本机与自动化验证阶段”“非生产验收” | “已部署 ELF2”“已完成真机/生产验收” |

品牌门户不能自动重定向到 V1 或 V2。用户应先理解版本与状态，再主动选择入口。所有跨域 CTA 都带状态副标题；`anima` 尚未开放时应显示可解释的开发状态页，而不是伪装成已经可用。

### 1.1 事实边界

站点文案仅使用当前仓库可以支持的事实：

- 代表作品为“基于 RK3588 的低时延多模态虚拟陪伴机器人”；
- V1 已证明开发板本地视觉、ASR、TTS、记忆、公网入口和 Live2D 链路可以连通，当前由 `robot.veyralux.org` 承载评审演示；
- V2 使用独立架构重写，已经有会话代际、记忆隔离、流式协议、连续情感、Live2D 和本地身体交互纵向实现与自动化测试；
- V2 当前只允许开发电脑和自动化环境验证，禁止替换 ELF2/公网 V1；
- 60 FPS、端到端时延、资源预算和真实手机体验仍是目标/验收门槛，不是已达成成绩；
- Strawberry Rabbit 模型公网再分发授权尚未闭环，品牌站不得直接打包或复用该模型资产作为装饰素材。

## 2. 品牌概念

### 2.1 核心命题

**让计算拥有持续的在场感。**

这句话表达的是研发方向，不声称系统已经具有人格、意识或医学级情感理解。辅助表述：

> VeyraLux 探索感知、记忆、语言与角色表现如何在端侧协同，让人机交互从一次指令，变成连续、可解释的陪伴体验。

### 2.2 视觉母题

建议视觉主题为 **“微光形成生命回路”**：

- 深色空间中存在一条柔和光带，随着滚动从单点分裂成感知、记忆、语言、表现四条轨迹；
- 光带不是炫技粒子雨，而是系统状态和章节进度的视觉隐喻；
- 图形语言使用半透明玻璃、微弱噪点、暖粉与琥珀高光、冷紫背景；
- 技术图示保持细线、明确箭头和真实状态标签，避免伪造“AI 大脑”或虚假实时数据；
- 未解决模型授权前，Hero 使用原创抽象图形、程序化光线或自有摄影，不出现 Strawberry Rabbit 模型文件、纹理或未经授权的宣传渲染。

建议颜色令牌：

## 3. 信息架构

采用“单页叙事首页 + 少量独立详情页”，避免把品牌门户做成控制台：

```text
/
├── #belief            品牌信念
├── #featured-work     代表作品
├── #system            系统协同
├── #experience        交互体验原则
├── #progress          V1 / V2 状态
├── #open-source       设计与开源
└── #contact           团队与入口

/work/companion-robot  代表作品详情（可后续加入）
/about                 VeyraLux / 微睿霖光与团队成员
/status                三个域名和版本状态说明
/legal                 隐私、第三方与素材授权说明
```

首版可以只实现首页、状态说明和 legal；作品详情先由首页长章节承载。导航建议：

- 左侧：VeyraLux 标志与“微睿霖光”；
- 中部：理念 / 作品 / 系统 / 进度；
- 右侧：`V1 演示` 次按钮、`V2 开发入口` 主按钮；
- 移动端：品牌标志、状态圆点和菜单按钮；展开后按章节列出，不隐藏版本标签。

### Screen 1：品牌信念 / Belief

**章节号**

> 01 / WHY WE BUILD

**主标题**

> 陪伴，不应从每一次唤醒重新开始

**正文**

> 一次正确回答不等于连续体验。真正困难的是在有限硬件上，让感知保持新鲜、让记忆可以修订、让旧任务能够被打断，也让角色的状态在沉默时依然连贯。

**三条原则**

1. **感知持续，回答取快照**

**动效分镜**

- 三条原则随滚动依次被一条光线连接；

### Screen 4：角色交互 / Experience

**主标题**

> 不给角色一块动作盘

**正文**

> 用户直接看向、触碰或与角色说话。动作由会话阶段、连续情感和模型能力共同决定；身体接触只产生短暂、可衰减的本地反射，不变成固定台词，也不上传原始指针轨迹。

**交互原则**

- **自主**：待机、倾听、思考和说话不依赖用户选择动作；
- **克制**：说话与倾听时降低接触反射，真实音频继续拥有口型控制权；
- **私密**：身体区域只转换成本地语义，坐标与路径不进入记忆；
- **可退出**：取消、重连和舞台销毁会清理旧反射与监听器。

**事实披露**

> 身体交互已进入 V2 本地实现和单元测试阶段，真实手机触控、误触率、无障碍和帧耗仍待验收。

**CTA**

目标：代码仓库中的 `new_code/docs/live2d-interaction.md`。

**动效分镜**

- 用原创抽象轮廓展示“注视、轻触、倾听”三种输入，不加载未授权模型；
- 光点接触轮廓后只产生一次 500–900 ms 弹性衰减；
- 不做跟随鼠标的大面积发光，也不采集/回传真实指针轨迹。

## 5. CTA 规则

### 5.1 文案层级

| 类型 | 用途 | 示例 |
| --- | --- | --- |
| Primary | 当前章节的唯一主动作 | `探索代表作品`、`查看 GitHub 仓库` |
| Secondary | 版本或资料入口 | `打开 V1 演示`、`阅读技术文档` |
| Status link | 尚未稳定的入口 | `V2 开发入口 · 功能可能变化` |
| Inline | 解释术语和边界 | `为什么 V2 暂不部署？` |

同一视口最多一个实心主 CTA。跳往 `robot` 或 `anima` 时显示目标域名和状态；不要把两个入口做成没有区别的“立即体验”。外部域名在新标签页打开时必须同时使用 `rel="noopener noreferrer"`，并在可访问名称中说明会打开新标签页。

### 5.2 状态徽标

统一使用：

- `演示中`：已有明确演示链路，但不等于生产服务；
- `开发中`：本机/自动化实现正在变化；
- `目标`：尚未有真实板端或真机验收；
- `待授权`：素材或模型不能进入公开制品。

禁止使用模糊的 `已上线` 同时描述 V1、V2 和品牌门户。

## 6. PC 与移动响应式规则

### 6.1 PC（主要宣传端）

- 基准视口 1440×900，内容最大宽度 1280 px，12 列网格，左右安全边距 `clamp(24px, 5vw, 80px)`；
- Hero 标题使用 `clamp(56px, 7.2vw, 112px)`，每行不超过 10 个中文全角字符的视觉宽度；
- 章节最小高度可以接近 100 svh，但内容超出时必须自然增长；
- 左侧章节号、中央主内容、右侧进度轨迹形成三层深度；
- 使用 sticky 章节可以增强叙事，但一次最多固定一个主要区域，不能把滚轮劫持成定长幻灯片；
- 鼠标 hover 只提供增强，所有 CTA 在无 hover 时仍有清晰边界和状态。

### 6.2 移动端

- 以 390×844 为基准，同时覆盖 320×568 与横屏 844×390；
- 内容边距 `clamp(18px, 5vw, 28px)`，主标题 `clamp(40px, 13vw, 64px)`；
- 不强制每屏 100vh；使用自然文档流，避免地址栏伸缩造成内容裁切；
- 导航、CTA 和底部内容加入 `env(safe-area-inset-*)`；
- 双列原则卡、V1/V2 卡和系统节点全部降为单列；
- 触摸目标至少 44×44 CSS px，主要 CTA 建议 48–56 px 高；
- 所有 hover 反馈改为 tap/focus 等价状态，视差幅度降至 PC 的 30% 以下；
- 横屏低高度时取消 sticky 和大标题占屏，优先保证文案与 CTA 可见；
- 不自动加载背景视频；使用 AVIF/WebP poster，用户主动播放后才请求媒体。

### 6.3 内容与布局不变量

- 320 px 宽度下无横向滚动；
- 中英文混排不截断域名、RK3588、Live2D 和 generation；
- 版本状态不能只靠颜色区分；
- 章节锚点考虑固定导航高度，跳转后标题不能被遮挡；
- 浏览器字体缩放到 200% 时不隐藏正文或 CTA。

| 层 | 内容 | PC 最大相对位移 | 移动端 |
| --- | --- | ---: | ---: |
| Back | 噪点、渐变、低亮光晕 | 视口高度 8% | 2% 或关闭 |
| Mid | 系统光带、章节编号 | 5% | 2% |
| Front | 标题/卡片 | 1–2% | 关闭 |

视差只使用 transform，不改变布局属性。不得让文字与背景产生无法阅读的持续相对运动。

### 7.2 章节跳转

- PC 右侧显示 01–08 纵向章节进度，当前章节同时显示编号和名称；
- 点击后使用原生锚点或可取消的平滑滚动，用户再次滚动立即接管；
- URL hash 同步当前章节，刷新和分享可恢复位置；
- 键盘 Tab 可以到达章节导航，Enter 激活；
- 移动端章节导航放在菜单中，不悬浮占据窄屏右侧；
- 禁止强制 scroll-snap，避免触控、键盘和辅助技术被困在某一屏。

### 7.3 上下页转场

品牌首页到作品、About、Status 等独立页面采用渐进增强：

1. 点击后立即给 CTA pressed/focus 状态；
2. 支持 View Transitions API 时，让品牌光带收束到目标页章节标题，持续 320–480 ms；
3. 不支持时使用正常导航，不引入等待遮罩；
4. 返回时恢复滚动位置，不重播完整 Hero；
5. 跨域进入 `robot`/`anima` 不做假装同站的页面 morph，只显示清晰的版本切换提示后正常打开。

### 7.4 动画时间语言

| 类型 | 时长 | 缓动 |
| --- | ---: | --- |
| 按钮/焦点反馈 | 120–180 ms | ease-out |
| 文字 reveal | 240–420 ms | cubic-bezier(0.22, 1, 0.36, 1) |
| 卡片/章节进入 | 320–560 ms | 同上 |
| 页面转场 | 320–480 ms | 标准 ease-in-out |
| 环境光循环 | 12–24 s | linear / alternate |

同一时刻不超过两个大面积元素运动。环境循环在标签页隐藏、低电量/省流或 reduced-motion 时停止。

## 8. 动效与加载性能预算

以下是发布门槛目标，不是当前已测成绩：

| 指标 | 目标预算 |
| --- | ---: |
| LCP | ≤2.5 s（移动 4G/中端机实验条件） |
| INP | ≤200 ms |
| CLS | ≤0.10 |
| 首屏关键 JS | ≤180 KiB gzip |
| 首屏 CSS | ≤70 KiB gzip |
| 首屏图片/图形 | 移动 ≤700 KiB，PC ≤1.2 MiB |
| 首屏总传输 | 移动 ≤1.0 MiB，PC ≤1.6 MiB，不含用户主动播放视频 |
| 动画帧预算 | 目标 16.7 ms/帧；P95 主线程动画工作 ≤8 ms |
| Long Task | 单次 <50 ms；滚动过程中不新增连续 Long Task |
| 同时提升为合成层 | 通常 ≤8 层，进入后及时释放 `will-change` |

实施规则：

- 首屏文字、导航和 CTA 由 HTML/CSS 首帧呈现，WebGL/Canvas 失败也不丢内容；
- 优先 CSS transform/opacity；复杂光带可使用一个降分辨率 Canvas，但不能每个章节各建渲染器；
- 不在 scroll handler 中读写交错布局；使用 rAF 合并、IntersectionObserver 激活章节；
- 不把 React/Preact 状态作为每帧动画时钟；
- 图片提供尺寸、AVIF/WebP 和响应式 `srcset`，首屏只 preload 一张关键视觉；
- 字体只预载一个必要子集，其余使用系统字体回退；
- 演示视频提供 H.264 MP4 与可选 WebM，默认 `preload="metadata"` 或 `none`；
- `Save-Data`、弱网或低性能设备直接切换静态背景；
- 所有预算以 Lighthouse、Performance trace 和真实移动设备复测，不能以开发机肉眼流畅替代。

## 9. `prefers-reduced-motion` 策略

当 `prefers-reduced-motion: reduce` 时：

- 禁用视差、光带循环、滚动绑定缩放、文字逐层 reveal 和页面 morph；
- 所有章节首帧直接处于最终可读状态；
- 章节跳转使用即时锚点，不做平滑滚动；
- 保留不位移的颜色/边框状态变化，时长不超过 120 ms；
- 视频不自动播放，Canvas 环境动画停止在高对比静态帧；
- 不能因为关闭动画而隐藏系统图节点、进度或 CTA。

站点设置中可以额外提供“减少动态效果”开关，但它只能比系统偏好更保守，不能默认覆盖用户已经选择的 reduced-motion。

## 10. 无障碍与隐私

### 10.1 无障碍

- 提供“跳到主要内容”链接，使用 `header/nav/main/section/footer` 语义结构；
- 每个章节只有一个清晰 `h2`，页面只有一个 `h1`；
- 正文与背景对比度至少 4.5:1，大标题至少 3:1；
- 可见焦点不能被品牌 glow 淹没，焦点环至少 2 px；
- 图标按钮都有中文可访问名称，状态徽标包含文字；
- 系统图提供等价文本列表，不能只让屏幕阅读器读取 SVG path；
- 视频提供字幕、文字稿和暂停按钮；自动循环视觉不得含每秒三次以上闪烁；
- 章节进度用 `aria-current` 表示当前项，不持续播报滚动百分比；
- 新标签页和跨域版本切换在链接名称中提前说明。

### 10.2 隐私

- 品牌门户不请求摄像头、麦克风、通知或精确位置权限；
- 不嵌入会自动连接 V1/V2 WebSocket 的隐藏 iframe；
- 分析工具只收集必要的聚合访问数据，并提供清晰说明；在需要同意的地区先取得 consent；
- 不记录细粒度鼠标轨迹、滚动录像或输入内容；
- 跨域 CTA 参数不得携带 session、用户 ID、密钥或 V1/V2 内部状态；
- 联系方式若使用表单，明确保存期限、用途和删除路径。

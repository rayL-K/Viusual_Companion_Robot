## Step 1：主交互页（改版前）— 不健康

### 可取之处

- ELF2 在线状态和公网地址可见。
- Live2D 舞台、回复、输入和工具区已经具备完整的信息结构。

### UX 与视觉问题

1. 输入操作网格发生横向溢出，“发送”按钮被裁切，核心任务无法可靠完成。
2. 底部工具依赖横向滚动，首屏只能看见部分入口，功能发现成本高。
3. 主舞台大量空白，而回复卡、输入区和工具区挤在底部，视觉重心失衡。
4. 在线徽标、回复卡和工具按钮的圆角、密度、层级不统一。
5. Live2D 显示 `Unknown error`，主角完全缺席，破坏产品核心体验。

### 无障碍风险

- 被裁切的发送按钮没有稳定触控区域。
- 横向滚动工具栏缺少明确的“还有更多”提示。
- 状态变化没有语义化状态区域；设备徽标也不是可识别的设置入口。
- 仅凭截图无法确认读屏顺序、键盘焦点和真机动态字体表现。

## 已实施改进

- 对所有按钮统一 `box-sizing`、最小宽度和网格约束，消除输入区溢出。
- 将工具入口改为四等分固定 Dock；设备、摄像头、历史和日志汇入“更多”面板。
- 设备状态改为可操作按钮；回复区声明为动态状态区域。
- 收紧间距、字号与短屏媒体查询，保留安全区。
- 升级官方 Live2D Cubism Core 5.1.0，使 MocVersion 5 模型可加载。

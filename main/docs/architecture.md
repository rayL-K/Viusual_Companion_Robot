# 架构说明

第一阶段目标是搭建一个简单、可替换的板端机器人循环：

```text
摄像头/麦克风 -> 感知模块 -> 对话大脑 -> 语音/动作决策 -> Live2D 显示
```

## 模块边界

- `devices`：硬件适配层，负责摄像头、麦克风、扬声器和显示设备的输入输出。
- `perception`：感知层，负责把图像和声音转成结构化结果，例如人脸位置、目标状态和语音识别文本。
- `brain`：对话与决策层，负责维护上下文、生成回复，并决定是否触发表情或动作。
- `speech`：语音输出层，负责文本转语音和音频播放。
- `ui.live2d`：虚拟形象层，负责 Live2D 模型状态、口型同步、眨眼、呼吸、表情映射和动作播放。
- `runtime`：运行时基础设施，负责进程生命周期、消息总线、日志和配置加载。
- `integrations`：模型与外部能力适配层，集中放置 RKNN、GGUF/llama.cpp、ONNX Runtime 和外部 API 封装。

## Live2D 方向

参考仓库：https://github.com/AkagawaTsurunaki/ZerolanLiveRobot

我们只借鉴高层设计，不复制实现：

- 角色控制必须独立于 ASR、TTS、LLM 等 AI 模块，避免一个模块阻塞整个显示层。
- 口型开合先由 TTS 音量或麦克风音量驱动，后续再升级为更精细的音素级口型。
- 待机动画、说话动画和情绪表情分开管理，避免动作互相覆盖。
- 显示层先抽象成通用显示表面，后续可以切换为开发窗口、VNC 调试画面或 HDMI 全屏展示。
